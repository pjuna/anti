# 🎓 AI/정보 교과 통합 수업 관리 및 포트폴리오 시스템

이 문서는 고등학교 인공지능/정보 교과 수업을 위해 맞춤 제작된 **통합 수업 관리 및 학생 포트폴리오 웹앱**의 개발 컨텍스트 및 아키텍처 가이드입니다. 새로운 개발 세션을 시작할 때 시스템의 전반적인 상태를 파악하기 위한 기준으로 사용됩니다.

---

## 📌 1. 프로젝트 개요 및 목표
* **목표**: 정보 교과 특성에 맞는 엄격한 과제 검증, 학생 포트폴리오 누적 관리, 그리고 교사의 학급 관리(출결/루브릭 평가)를 하나의 플랫폼으로 통합.
* **주요 특징**:
  * **보안 및 프라이버시**: 개인정보 수집 최소화 및 `pg_cron`을 활용한 기간 만료 데이터 자동 파기.
  * **과제 진실성 검증**: 학생 보고서 본문의 코드와 실제 제출 소스코드 간 일치 여부를 자동 검증(Strict Verify).
  * **오류 방지**: 코딩 초보자가 자주 하는 '스마트 따옴표' 실수를 자동 정규화.
  * **다이나믹 관리**: 시스템 약관, 보관 기간, 가입 코드를 DB화하여 코드 수정 없이 관리자 페이지에서 즉시 제어.

## 🛠 2. 기술 스택 (Tech Stack)
* **Frontend / Framework**: `Streamlit` (Python 기반 반응형 웹 UI)
* **Backend / Database**: `Supabase` (PostgreSQL 기반 BaaS)
  * **Auth**: 사용자 인증 및 세션 관리
  * **Database**: 테이블 및 RLS(Row Level Security)를 통한 행 단위 접근 제어
  * **Storage**: 제출물 파일(PDF, HWP 등) 저장 버킷 (`submissions`)
  * **pg_cron**: 백그라운드 데이터 자동 정리 스케줄러

## 🗄 3. 데이터베이스 아키텍처 (Supabase DB)

모든 스키마 및 보안 정책은 `schema.sql`에 정의되어 있습니다.

### 핵심 테이블 구조
1. **`profiles`**: Auth.users와 1:1 매칭. 이름, 역할(admin/teacher/student), 생년월일, 전화번호 저장.
2. **`signup_codes`**: 무분별한 가입을 막기 위해 관리자가 배포하는 '그룹 가입 코드'.
3. **`classes`**: 교사가 개설하는 수업 인스턴스. 연도/학기를 구분하며 6자리 랜덤 `join_code` 발급.
4. **`assignments`**: 루브릭 데이터(JSON)를 포함한 과제 정보.
5. **`submissions`**: 학생의 과제 제출물. 텍스트 보고서, 원본 소스코드, 파일 다운로드 URL, 코드 검증 상태(`is_verified`), 최종 점수 포함.
6. **`attendance`**: 수업별/날짜별 학생 출결 상태.
7. **`settings`**: 관리자가 제어하는 동적 설정값 (개인정보 약관, 동의 거부 메시지, 데이터 보관 연수).

### 주요 보안 및 자동화 로직
* **RLS (Row Level Security)**: 학생은 오직 자신의 제출물과 프로필만 조회/수정할 수 있도록 데이터베이스 단에서 완벽히 격리됩니다. 스토리지 파일에도 동일한 RLS가 적용됩니다.
* **데이터 연쇄 삭제 (CASCADE)**: 사용자가 탈퇴/삭제되면 관련된 출결, 과제 데이터가 함께 삭제됩니다.
* **개인정보 자동 파기 (`pg_cron`)**: 매일 자정에 `settings` 테이블의 `data_retention_years` 값을 참조하여 보관 기간이 지난 유저(auth.users)를 데이터베이스 내부에서 스스로 삭제합니다.

## ⚙️ 4. 구현된 핵심 비즈니스 로직 및 기능
* **세션 지속성 (Persistent Login)**: 브라우저 쿠키를 활용하여 새로고침 시에도 로그인 상태를 자동으로 복구합니다.
* **이원화된 가입 로직**: 학생은 코드 없이(학년/반/번호 입력), 교사/관리자는 전용 코드를 사용하여 가입하는 이원화된 프로세스를 제공합니다.
* **모듈화된 시스템 설정**: 관리자 페이지에서 약관, 보관 주기, 학적 범위를 탭별로 동적 관리할 수 있습니다.
* **프리미엄 UI/UX**: 그라데이션 프로필 카드 및 블록 스타일의 사이드바 내비게이션을 적용했습니다.
* **스마트 따옴표 정규화 (`utils/db.py`)**: 특수 따옴표를 표준으로 자동 변환합니다.
* **Strict Verify 교차 검증 (`utils/db.py`)**: 보고서 내 코드 블록과 실제 소스코드 간 일치 여부를 검증합니다.
* **실시간 유효성 검사 (`app.py`)**: 회원가입 시 비밀번호 규칙 등을 실시간으로 피드백합니다.

## 🚀 5. 향후 구현 로직 및 다음 단계 (Roadmap)

다음 개발 세션에서는 아래 [Phase 3]의 '채점 센터'부터 이어서 작업하면 됩니다.

* **[Phase 3] 교사용 '제출물 채점 센터' (최우선 과제)**:
  * 특정 수업 및 과제를 선택하면 제출한 학생 목록 출력.
  * 학생 선택 시 제출물(보고서, 코드, 파일) 상세 뷰 제공 및 **Strict Verify 결과 하이라이트**.
  * 설정된 **루브릭** 항목별로 점수를 입력하고 총점을 자동 합산하여 `submissions` 테이블에 업데이트.
* **[Phase 4] 학생 마이 포트폴리오 시각화**:
  * 단순 리스트를 넘어, 누적된 과제와 교사의 피드백을 타임라인 또는 차트 형태로 시각화하여 학기말 성찰 자료로 활용하도록 구성.
* **[Phase 3] 출결 데이터 시각화**:
  * 현재의 일별 체크 기능을 넘어, 특정 학생의 전체 결석/지각 횟수를 집계하는 대시보드 추가.
* **[Phase 3] CSV Export 정교화**:
  * 채점 데이터를 학교 시스템(나이스)에 입력하기 좋은 형태로 가공하여 다운로드.

## 📁 6. 주요 파일 구조
```text
/
├── app.py                  # 메인 앱 엔트리 (인증, 라우팅, 실시간 회원가입 로직)

├── utils/
│   └── db.py               # Supabase 클라이언트 연결, 코드 검증, Storage 업로드 함수
├── views/
│   ├── admin_view.py       # 최고관리자 대시보드 (가입 코드 관리, 시스템 설정 관리)

└── .streamlit/
    └── secrets.toml        # (Git 제외) Supabase URL 및 API Key 환경변수
```
